## Project

Portfolio website for Savely Danko. Static SPA (React 19 + Vite) with EN/RU i18n, deployed on a VPS via Ansible. Includes a CMS admin panel.

## Stack

- **Frontend**: React 19, TypeScript, Vite, Tailwind CSS v4, Framer Motion, React Router v7
- **Admin backend**: Node.js, Express, JWT (HttpOnly cookie), bcryptjs, tsx
- **Deploy**: Ansible → Ubuntu VPS, Nginx, Certbot (Let's Encrypt), systemd

## Key directories

```
src/                        # Main site source
  app/                      # App root, providers, SiteMeta, routing
  pages/                    # One file per route (thin orchestrators)
  sections/                 # Large page blocks (Hero, About, Projects…)
  components/ui/            # Design primitives
  components/layout/        # Header, Footer, Section wrapper
  data/                     # Content: projects.ts, blog.ts, aboutPage.ts…
  i18n/en.ts, i18n/ru.ts   # All UI strings + privacy policy text
  config/site.ts            # siteConfig (name, URL, base path)
  constants/                # navigation.ts, socials.ts

admin/
  backend/                  # Express API (port 3001)
    server.ts               # Entry point — app.set("trust proxy", 1) required for nginx
    routes/                 # auth, projects, blog, publish
    data/                   # projects.json, blog.json — runtime data store
  frontend/                 # Admin SPA (Vite, base: "/admin/")
    src/pages/              # Login, ProjectsPage, BlogPage
    src/components/forms/   # ProjectForm, BlogArticleForm
    src/lib/api.ts          # Typed fetch client with credentials: "include"

ansible/
  group_vars/vps.yml        # All deploy variables incl. admin_password_hash, admin_jwt_secret
  roles/webapp/
    tasks/main.yml          # Full deploy sequence
    templates/nginx.conf.j2 # Nginx config — uses alias + ^~ for /admin/, /api/
```

## Critical rules

1. **Never edit `src/data/projects.ts` or `src/data/blog.ts` directly** when the admin panel is in use — these files are regenerated by `POST /api/publish`. Edit via the admin UI or `admin/backend/data/*.json`.
2. **i18n**: UI text lives in `src/i18n/en.ts` and `src/i18n/ru.ts`. Both files must stay in sync (same keys). The `Translations` type is derived from `en.ts`.
3. **Privacy policy** is in `src/i18n/*.ts` under the `privacy` key — not a separate data file.
4. **Admin cookie** uses `SameSite=Strict; Secure; HttpOnly`. The backend needs `app.set("trust proxy", 1)` to detect HTTPS behind nginx.
5. **Nginx** serves `/admin/*` via `alias` + `location ^~ /admin` with a named `@admin_fallback` for SPA routing. The `^~` modifier is required so the regex asset-caching location does not intercept admin JS/CSS.

## Adding a new page

1. Create `src/pages/NewPage.tsx` and export it from `src/pages/index.ts`
2. Add a lazy import + `<Route>` in `src/app/App.tsx`
3. Add meta in `src/app/SiteMeta.tsx`
4. Add i18n keys to `src/i18n/en.ts` and `src/i18n/ru.ts`
