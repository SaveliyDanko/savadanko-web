# Architecture

## Стек

- **React 19** — UI-библиотека
- **TypeScript** — типизация
- **Vite** — сборщик
- **Tailwind CSS v4** — стили (утилитарный подход)
- **Framer Motion** — анимации
- **Lucide React** — иконки
- **React Router DOM** — маршрутизация (подключена, но пока не задействована — сайт одностраничный)

## Архитектурный подход

**Feature-Sliced Light** — облегчённая модульная архитектура с чётким разделением ответственности, без лишних абстракций.

Ключевые принципы:
- Каждый слой отвечает за свою задачу
- Компоненты изолированы и переиспользуемы
- Данные отделены от отображения
- Импорты предсказуемы через `@/` alias

## Назначение директорий

### `app/`
**Что:** корневой компонент, глобальные провайдеры, конфиг роутера.
**Зачем:** единая точка сборки приложения — все глобальные обёртки здесь.
**Нельзя:** бизнес-логику, UI-компоненты.

### `pages/`
**Что:** страницы-оркестраторы, которые собирают секции в нужном порядке.
**Зачем:** тонкий слой между роутером и контентом.
**Нельзя:** сложную логику или стили — всё делегируется секциям и компонентам.

### `sections/`
**Что:** крупные самодостаточные блоки страницы (Hero, About, Projects...).
**Зачем:** каждая секция — изолированный модуль со своей разметкой и логикой.
**Нельзя:** переиспользуемые компоненты (они идут в `components/`).

### `components/ui/`
**Что:** базовые UI-примитивы (Button, Card, Badge, SectionHeading).
**Зачем:** единообразный дизайн-язык проекта.
**Нельзя:** компоненты с доменной логикой.

### `components/layout/`
**Что:** каркас страницы — Header, Footer, Section-обёртка.
**Зачем:** структура, которая не меняется между страницами.
**Нельзя:** контент или бизнес-логику.

### `components/common/`
**Что:** доменные переиспользуемые компоненты (ProjectCard, TimelineItem).
**Зачем:** блоки, которые используются в нескольких секциях.
**Нельзя:** чистые UI-примитивы (они в `ui/`).

### `hooks/`
**Что:** кастомные React-хуки (useScrollSpy, useMediaQuery).
**Зачем:** инкапсуляция переиспользуемой логики.
**Нельзя:** компоненты, утилиты без React-зависимости.

### `utils/`
**Что:** чистые функции без React-зависимости (cn, форматирование).
**Зачем:** вспомогательная логика, доступная из любого слоя.
**Нельзя:** хуки, компоненты, функции с side-effects.

### `types/`
**Что:** общие TypeScript-интерфейсы и типы.
**Зачем:** единые контракты данных для всего проекта.
**Нельзя:** типы, специфичные для одного компонента (они рядом с компонентом).

### `constants/`
**Что:** статические конфигурации (навигация, соцсети).
**Зачем:** данные, которые не меняются в runtime.
**Нельзя:** динамические данные, контент проектов.

### `data/`
**Что:** контентные данные (проекты, опыт, навыки).
**Зачем:** отделение контента от компонентов — легко обновлять без правки JSX.
**Нельзя:** конфигурации UI, константы навигации.

### `assets/`
**Что:** изображения, шрифты, SVG-иконки.
**Зачем:** централизованное хранение статических ресурсов.
**Нельзя:** код, стили, данные.

### `styles/`
**Что:** глобальный CSS, Tailwind-тема, base-стили.
**Зачем:** единая точка конфигурации визуальной темы.
**Нельзя:** компонентные стили (используйте Tailwind-классы в JSX).

## Как вносить изменения

| Задача | Где делать |
|---|---|
| Новая страница | `pages/NewPage.tsx` + роут в `app/` |
| Новая секция | `sections/NewSection/NewSection.tsx` + `index.ts` |
| Новый UI-примитив | `components/ui/Component.tsx` + экспорт в `index.ts` |
| Новый доменный компонент | `components/common/Component.tsx` + экспорт в `index.ts` |
| Новый хук | `hooks/useHookName.ts` + экспорт в `index.ts` |
| Новая утилита | `utils/utilName.ts` + экспорт в `index.ts` |
| Новый тип | `types/index.ts` |
| Изменить контент | Файлы в `data/` |
| Изменить тему | `styles/globals.css` → блок `@theme` |

## Именование

- **Компоненты:** `PascalCase.tsx` (Button.tsx, ProjectCard.tsx)
- **Хуки:** `camelCase.ts` с префиксом `use` (useScrollSpy.ts)
- **Утилиты:** `camelCase.ts` (cn.ts, formatDate.ts)
- **Данные/константы:** `camelCase.ts` (projects.ts, navigation.ts)
- **Типы:** `PascalCase` для интерфейсов, `camelCase` для переменных типов
- **Папки секций:** `PascalCase` (Hero/, About/, Projects/)
- **Barrel-экспорты:** каждая папка с `index.ts` для чистых импортов

## Path Alias

```ts
import { Button } from "@/components/ui";
import { useScrollSpy } from "@/hooks";
import type { Project } from "@/types";
```

Настроен через `vite.config.ts` (resolve.alias) и `tsconfig.app.json` (paths).
