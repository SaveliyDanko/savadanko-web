# Как создавать и обновлять страницы блога

Эта памятка описывает текущую систему блога в проекте.

## Где находится логика блога

Основные файлы:

- `src/data/blog.ts` — единый реестр всех статей
- `src/pages/BlogPage.tsx` — список статей на странице `/blog`
- `src/pages/BlogArticlePage.tsx` — отдельная страница статьи `/blog/:slug`
- `src/app/App.tsx` — маршруты блога

## Как устроена система

Каждая статья хранится как объект в массиве `blogArticles` в `src/data/blog.ts`.

У каждой статьи есть:

- `slug` — уникальный адрес статьи
- `tags` — теги карточки и страницы статьи
- `content.en` — английская версия статьи
- `content.ru` — русская версия статьи

Маршрут формируется автоматически по `slug`:

- `slug: "article-1"` -> `/blog/article-1`
- `slug: "my-first-post"` -> `/blog/my-first-post`

Карточка на странице `/blog` и полная страница статьи используют один и тот же объект из `blogArticles`.

## Как добавить новую статью

1. Открой `src/data/blog.ts`.
2. Скопируй существующий объект внутри массива `blogArticles`.
3. Вставь копию ниже и замени значения.

Минимальный шаблон:

```ts
{
  slug: "article-2",
  tags: ["react", "frontend", "architecture"],
  content: {
    en: {
      label: "Article",
      title: "My second article",
      excerpt: "Short description for the blog card and article header.",
      publishedAt: "April 10, 2026",
      readingTime: "4 min read",
      lead: "Opening paragraph of the full article page.",
      sections: [
        {
          title: "Section title",
          paragraphs: [
            "First paragraph.",
            "Second paragraph.",
          ],
          bullets: [
            "Optional bullet 1",
            "Optional bullet 2",
          ],
        },
      ],
    },
    ru: {
      label: "Статья",
      title: "Моя вторая статья",
      excerpt: "Короткое описание для карточки и заголовка статьи.",
      publishedAt: "10 апреля 2026",
      readingTime: "4 мин чтения",
      lead: "Вступительный абзац полной страницы статьи.",
      sections: [
        {
          title: "Название секции",
          paragraphs: [
            "Первый абзац.",
            "Второй абзац.",
          ],
          bullets: [
            "Необязательный пункт 1",
            "Необязательный пункт 2",
          ],
        },
      ],
    },
  },
}
```

4. Укажи уникальный `slug`.
5. Заполни `en` и `ru` версии.
6. Сохрани файл.

После этого статья автоматически:

- появится в списке на `/blog`
- станет кликабельной карточкой
- откроется по адресу `/blog/<slug>`

## Какие поля за что отвечают

`label`
: короткая плашка над заголовком карточки и статьи

`title`
: заголовок карточки и страницы статьи

`excerpt`
: описание в карточке и под заголовком статьи

`publishedAt`
: дата публикации

`readingTime`
: примерное время чтения

`lead`
: первый выделенный текстовый блок на странице статьи

`sections`
: основной контент статьи

`sections[].title`
: заголовок секции

`sections[].paragraphs`
: обычные абзацы секции

`sections[].bullets`
: необязательный список пунктов

## Как обновить существующую статью

1. Найди статью по `slug` в `src/data/blog.ts`.
2. Обнови нужные поля в `content.en` и `content.ru`.
3. Если меняешь `slug`, изменится и URL статьи.
4. Если меняешь только текст, карточка и страница обновятся автоматически.

## Рекомендации по slug

Используй короткие URL-safe значения:

- `article-1`
- `react-performance-notes`
- `how-i-built-portfolio-blog`

Избегай:

- пробелов
- кириллицы в `slug`
- дубликатов уже существующих `slug`

## Что уже реализовано автоматически

Ничего дополнительно подключать не нужно:

- маршрут `/blog/:slug` уже настроен
- страница списка уже использует `blogArticles`
- страница статьи уже умеет искать публикацию по `slug`
- если `slug` не найден, показывается `404`

## Быстрый чек перед коммитом

Перед коммитом полезно проверить:

1. У статьи уникальный `slug`
2. Заполнены обе локали: `en` и `ru`
3. Нет пустых `paragraphs`
4. Карточка открывает правильный URL `/blog/<slug>`

## Если захочешь развить систему дальше

Следующими шагами можно добавить:

- обложки статей
- отдельное SEO-описание для каждой статьи
- related posts
- markdown/MDX-авторинг
- черновики и флаг публикации
